## Overall Impression

Solid, consistent homepage, but two real bugs underneath: a body-text color that fails WCAG AA almost everywhere it's used, and a motion-preference rule that's backwards (adds motion for reduced-motion users, gives none to default users). Both contradict PRODUCT.md's own stated commitments.

## What's Working
- Work grid now has real links and specific alt text (fixed earlier this session).
- "Book a meeting" confirmed functional via script.js fallback to contact.html#start-project.
- Pricing copy is specific and transparent, matching PRODUCT.md's stated principle.

## Priority Issues

[P1] .section-subhead (color: var(--color-grey) #767c7e, 1.125rem regular — not "large text" under WCAG) renders at 3.88:1 against the page background (#f0f6f8), failing the 4.5:1 body-text minimum. This is the standard secondary-paragraph class sitewide, including the entire privacy.html policy body. Fix: darken --color-grey for body-weight text, or split into a separate darker token. Suggested command: /impeccable colorize.

[P1] Same grey fails worse inside lighter service/pricing card panels. Computed contrast: black bg 4.5:1 (passes barely), #23272d 3.54:1 (passes only because text is large), #3a3e44 2.54:1 (fails), #50555a 1.78:1 (fails badly). The Custom tier's price ("from £2,500") sits on the 2.54:1 panel — the single most important text for cost comparison. Fix: lighten text on the two lightest panel steps, or adjust the panel ramp. Suggested command: /impeccable colorize.

[P2] prefers-reduced-motion logic inverted for .service-tags li: the only hover/focus CSS (scale + bounce easing) is scoped inside @media (prefers-reduced-motion: reduce), while the JS magnetic effect is correctly gated to fire only with no preference. Net effect: default users get rich motion, reduced-motion users get a bounce transform they shouldn't. Fix: swap the media query to no-preference. Suggested command: direct CSS fix.

[P3] Decorative hero wordmark SVG has no aria-hidden, redundant with the header's already-announced "Omiko" text link. Sitewide pattern. Fix: add aria-hidden="true". Suggested command: direct HTML fix.

## Persona Red Flags

Sam (Accessibility-Dependent User): needs 4.5:1 contrast and respected reduced-motion; both fail in measurable ways here.

Casey (Distracted Mobile User, primary PRODUCT.md persona): low contrast is exactly the failure mode for outdoor phone viewing; the Custom tier's price is the textbook case.
